# AI 决策系统扩展结构设计

本文档用于将新增 AI 决策系统需求映射到当前项目结构，确保后续每一步开发都能在不打乱现有页面体系与视觉语言的前提下逐步落地，并在每一步验证后提交到 GitHub。

| 模块目标 | 对应页面或文件 | 计划实现方式 | 验证方式 |
| --- | --- | --- | --- |
| 独立 AI 决策页面 | `client/src/pages/`、`client/src/App.tsx`、现有侧边导航组件 | 新增单独页面与左侧标签入口，保持现有深色平台风格与导航结构一致 | 路由跳转、侧边菜单高亮、页面加载验证 |
| 情景模拟与预测输入 | 新 AI 页面、`server/routers.ts`、相关数据模块 | 增加月份、价格、产能、需求等输入，返回利润、成本、均价、资源测算与曲线数据 | 前后端联调、输入输出一致性验证 |
| 多 Agent 决策层 | `server/routers.ts`、新的服务端决策模块 | 输出全局决策层、业务调度层、现场执行层的建议、目标、动作与优先级 | tRPC 返回结构验证、页面展示验证 |
| 红黄绿预警与九宫格 | 新 AI 页面与告警数据接口 | 基于阈值结果生成红黄绿状态，并映射到九宫格卡片与告警面板 | 阈值切换验证、页面状态显示验证 |
| 根因分析弹窗 | 新 AI 页面与 AI/规则分析接口 | 异常时打开弹窗，展示问题类型、影响范围、预计损失、建议动作与根因 | 弹窗交互验证、数据字段完整性验证 |
| 自动派单与执行反馈 | 新调度接口、工单数据结构、前端状态面板 | 生成标准化 JSON 工单，展示厂长、司机、仓储管理员等角色的执行状态与升级预警 | 工单字段验证、状态流转验证 |
| 移动端与工业平板适配 | 新 AI 页面响应式布局 | 保持桌面端大屏信息密度，同时提供窄屏卡片堆叠布局 | 浏览器响应式预览验证 |
| 分步提交要求 | `todo.md` 与 Git 历史 | 每一步完成后先验证，再提交并推送到 GitHub，然后进入下一步 | Git 提交记录与远程推送成功 |

后续开发顺序将严格遵循：先新增导航与独立页面，再补充情景模拟与多 Agent 数据模型，再完成告警、根因分析与派单反馈，最后补齐测试与端到端验证。每一阶段完成后，都需要先执行对应验证，再提交并推送到 GitHub。
