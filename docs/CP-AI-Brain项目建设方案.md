# CP-AI Brain 项目建设方案

版本：V1.0  
制定日期：2026-04-24  
适用对象：项目负责人、产品经理、算法工程师、前后端工程师、数据工程师、业务战房、财务、供应链、工厂和仓储执行团队。

## 1. 建设目标

CP-AI Brain 的建设目标是把猪事业全产业链从“展示型战房”升级为“实时量化决策与执行闭环平台”。平台要能够持续接入真实业务数据，自动计算多种套利与调度策略，输出可解释的 AI 建议，并通过派单、回执、审计和复盘机制推动现场执行。

目标分三层：

1. 业务目标：减少低价周期亏损，提升部位、库存、产能、区域和资金的综合收益。
2. 系统目标：形成数据采集、算法计算、AI 决策、执行调度、审计复盘的闭环平台。
3. 组织目标：让管理层、业务调度、工厂、仓储、物流、销售和财务共用同一套数据口径和决策标准。

## 2. 建设原则

1. 数据优先：没有可信数据不做自动决策，只做模拟和提示。
2. 公式可解释：所有关键建议都必须能回溯到成本、收益、约束、阈值和风险分。
3. 人机协同：AI 负责计算、归因、建议和派单，高风险动作必须保留人工审批。
4. 小步闭环：每个阶段都要有可运行页面、可验证接口、可落库记录和可复盘结果。
5. 业务口径先行：财务成本、产能、库存、价格、订单、损耗等口径必须由业务负责人确认。
6. 演示数据和真实数据分离：所有模拟、fallback、seed 数据必须显式标记，避免误用。
7. 安全合规：权限分级、操作留痕、敏感数据脱敏、金融策略合规审查必须前置。

## 3. 目标架构

建议目标架构分为 8 层：

```text
展示交互层
  大屏战房 / 桌面工作台 / 移动端角色页 / 运营看板

应用服务层
  tRPC/API 网关 / 业务编排 / 审批流 / 工单流 / 通知流

算法决策层
  时间套利 / 空间套利 / 金融套利 / 深度套利 / 全局优化 / 风险评分

AI 智能层
  LLM 结构化建议 / Agent 分工 / 根因分析 / 自然语言调参 / 决策解释

数据服务层
  行情服务 / 库存服务 / 产能服务 / 成本服务 / 订单服务 / 财务服务

数据中台层
  ODS / DWD / DWS / ADS / 指标口径 / 数据质量 / 数据血缘

集成对接层
  SAP / WMS / TMS / MES / 战房 / UWORK / 外部行情 / 期货 / 企业微信 / 短信

基础设施层
  MySQL / Redis / 对象存储 / 队列 / 任务调度 / 监控告警 / CI/CD
```

当前代码已经覆盖展示交互层、应用服务层、部分算法决策层、部分 AI 智能层和 MySQL 持久化。后续重点是补强数据服务层、数据中台层、集成对接层和生产运维层。

## 4. 产品模块规划

### 4.1 全链总览

目标：为管理层提供全链经营态势。

功能：

- 出栏、饲料、屠宰、分割、速冻、仓储、物流、深加工、销售核心指标。
- 事业部卡片：猪、鸡、饲料等。
- 经营风险：价格跌破成本、库存超龄、产能瓶颈、订单缺口、资金占用。
- 关键动作入口：进入猪事业、AI 决策、量化决策、套利页面。

下一步：

- 将 `platform.snapshot` 从静态 seed 升级为真实指标聚合。
- 为每个指标增加数据来源、更新时间、责任部门和可信度。

### 4.2 猪事业行情与 23 部位经营

目标：以 23 部位为最小经营单元，形成行情、成本、库存、渠道、期货映射和策略入口。

功能：

- 生猪、白条、冻品、玉米、豆粕、期货主力合约行情。
- 23 部位现货价、冻品价、期货映射价、预测价、基差、涨跌幅。
- 部位分类、搜索、排序、价格曲线、移动均线和视窗控制。
- 库存批次、库龄、成本、当前价和期货映射价。
- 从部位或批次进入 AI 决策和量化策略。

下一步：

- 建立部位主数据表和价格口径表。
- 接入真实 WMS 库存、财务成本和销售价格。
- 对外部行情源建立数据质量和缺口告警。

### 4.3 量化决策引擎

目标：对库存批次给出持有或出售建议。

功能：

- 输入批次和角色。
- 输出 1、2、3 个月方案。
- 计算保本价、未来持有成本、预计售价、净收益、风险分和风险等级。
- 高风险策略提交审批，普通策略确认执行。
- 写入审计日志。

下一步：

- 增加更多持有周期：周、月、季度、半年、年。
- 引入真实部位损耗率、资金成本、冷库成本和合同约束。
- 增加策略回测：建议收益与实际执行收益对账。

### 4.4 AI 决策工作台

目标：成为战房日常决策和执行指挥主页面。

功能：

- 价格预测：真实历史、预测曲线、保本线、平均售价、公斤利润。
- What-If：价格、产能、需求变化下的资源计划。
- 多 Agent：总部经营、业务调度、现场执行。
- 红黄绿预警：利润偏差、产能利用、仓储压力、冷链时效、需求波动、执行节奏等。
- 根因分析弹窗：影响范围、预计损失、AI 建议、责任人、根因。
- 自动派单：厂长、司机、仓储管理员工单。
- 执行反馈：确认、执行中、完成、超时升级。
- 移动端角色视图：现场角色可直接操作。

下一步：

- 把派单对象从模拟角色扩展为真实组织、人、设备和班次。
- 增加审批流：高风险动作、超额库存、跨区调拨、金融套保。
- 增加工单 SLA、超时升级规则、重复提醒和责任追踪。

### 4.5 时间套利

目标：在猪价低于成本时计算收储量、持有周期和释放窗口。

功能：

- 输入毛猪价、社会养殖成本、持有成本、收储吨数、起始月和持有时长。
- 输出持有成本、未来价格、利润空间、套利窗口、最佳释放月。
- 结合屠宰、分割、速冻、仓储、深加工和销售产能。
- 支持多方案保存和对比。

下一步：

- 使用真实期货曲线、现货远期报价和历史季节因子。
- 引入库龄 FEFO、库存批次、冷库容量、合同订单和资金上限。
- 将启发式产能计划升级为约束优化模型。

### 4.6 空间套利

目标：用区域价差覆盖冷链、损耗和机会成本，形成跨区调拨策略。

功能：

- 全国省份产地和销地节点。
- 产地价格、销地价格、距离、运费、净利、批次总利润。
- 小型、中型、大型冷链车型自动选择。
- 产能、需求、目标发运量约束。
- 鲜销、冻储、深加工三类利润通道分配。
- AI 岗位任务：采购、物流、销售、风控。

下一步：

- 接入真实 TMS 路线、车辆、司机、油价、过路费、温控和损耗。
- 增加多仓、多中转、时窗、装载率和回程配载。
- 使用网络流或 MILP 求解替代纯贪心。

### 4.7 金融套利

目标：为库存和现货敞口提供期现、基差、套保和资金风险分析。

功能：

- 输入现货、期货、未来价格、现货敞口、套保比例、保证金率、合约规模。
- 输出现货损益、期货损益、基差损益、资金成本、仓储成本、交易成本、净收益。
- 输出保证金、资金占用、追保风险、套保有效性。
- 输出敏感性和压力测试。

下一步：

- 明确真实使用边界：该模块生产使用必须经过财务和合规审批。
- 接入期货账户、持仓、保证金、合约日历和风控限额。
- 增加策略审批、风险限额、止损规则和每日盯市。

### 4.8 深度套利

目标：从单一冻储套利升级为全产业链机会识别。

功能：

- 14 类深度套利机会识别。
- 输入玉米、豆粕、生猪价、能繁母猪、产能利用率、库龄、冷链、仓储、资金、区域价差、政策和品牌认证。
- 输出机会状态、触发条件、预期收益、风险等级、执行建议和指标。

下一步：

- 每类套利建立历史样本、阈值校准和回测报告。
- 增加机会优先级：收益、风险、执行难度、资金占用、组织可控性。
- 与自动派单系统打通，形成“机会 - 策略 - 工单 - 收益复盘”闭环。

### 4.9 全局最优化调度

目标：统一求解屠宰、分割、冻储、销售、运输和深加工的最优组合。

功能：

- 输入出栏、出品率、产能、成本、订单、深加工需求和运输成本。
- 输出利润、销售、分割、生产、库存、运输等表。
- 输出总收入、总成本、总利润、利润率、屠宰量、销售量、冻储量、头均利润、产能利用率。
- 支持调参、策略预设、批量场景对比、AI 自然语言调参。

下一步：

- 明确优化目标：利润最大、减亏最大、风险调整收益最大、现金流优先、库存周转优先。
- 建立硬约束：产能、库容、资金、合同、订单、时窗、食品安全、合规。
- 建立软约束和惩罚：风险、执行复杂度、库龄、损耗、调拨距离、客户优先级。
- 引入 OR-Tools、Pyomo、HiGHS、Gurobi 或 CPLEX 等专业求解器。

### 4.10 审计与治理

目标：保证所有关键决策可追溯、可审批、可复盘。

功能：

- 策略确认审计。
- 高风险审批审计。
- AI 派单生成审计。
- 工单回执更新审计。
- 超时升级审计。
- 套利记录展示。

下一步：

- 增加审批流状态机。
- 增加审计查询筛选：角色、风险、实体、时间、动作类型。
- 增加差异对账：建议收益、执行收益、偏差、原因、责任部门。

## 5. 数据建设方案

### 5.1 数据域划分

建议按 10 个数据域建设：

1. 主数据域：猪场、屠宰厂、冷库、产线、车辆、客户、供应商、部位、人员、组织。
2. 养殖域：存栏、出栏、均重、健康、饲料、疫病、防疫、养殖成本。
3. 屠宰分割域：屠宰计划、实际屠宰、分割出品、部位出品率、人工、设备、班次。
4. 速冻仓储域：速冻产能、入库、出库、库龄、库容、温控、损耗、FEFO。
5. 物流冷链域：路线、车辆、司机、运费、油费、时效、温控、在途损耗。
6. 深加工域：产线、配方、投料、产出、订单、加工成本、成品售价。
7. 销售渠道域：客户、订单、渠道价、批发价、商超价、餐饮价、电商价、合同。
8. 市场行情域：现货、期货、基差、原料、竞品、区域供需、政策信号。
9. 财务资金域：成本、费用、资金利率、库存价值、应收账款、现金流、授信。
10. 执行审计域：工单、回执、通知、审批、操作日志、执行收益。

### 5.2 数据分层

建议数据中台采用：

- ODS：原始接入层，保留源系统字段。
- DWD：明细清洗层，统一主键、单位、时间、地区、部位和组织。
- DWS：主题汇总层，按日/小时/分钟形成经营宽表。
- ADS：应用服务层，为页面、算法和 AI 提供直接消费数据。
- Metric Store：指标口径层，统一毛利、保本价、全成本、库龄、产能利用率等指标。

### 5.3 数据质量规则

必须建立的数据质量规则：

- 唯一性：批次号、工单号、部位编码、工厂编码不能重复。
- 完整性：关键字段如价格、数量、单位、时间、地点、负责人不能为空。
- 合法性：价格、吨数、头数、成本不能为负；枚举值必须在字典内。
- 一致性：吨/kg、元/kg、元/吨、头/kg 的单位换算统一。
- 及时性：行情分钟级，库存小时级，产能日级，财务日/月级。
- 对账：库存数量与 WMS 对账，销售与 ERP 对账，成本与财务对账。
- 异常：价格跳变、库存突增突减、工单超时、库龄超阈值自动告警。

### 5.4 真实数据接入优先级

第一优先级：

- 生猪、白条、冻品、玉米、豆粕、期货行情。
- 库存批次、部位、数量、库龄、成本。
- 屠宰、分割、速冻、仓储产能。
- 销售订单和渠道价格。
- 财务全成本和资金成本。

第二优先级：

- 冷链车辆、路线、温控和损耗。
- 深加工投料、产线和成品售价。
- 供应商、客户和合同约束。
- 质检和食品安全数据。

第三优先级：

- IoT 设备、视觉识别、自动化产线。
- 跨境、政策、保险、期权和外部宏观数据。

## 6. 算法建设方案

### 6.1 算法分级

建议按 L1-L5 建设：

| 等级 | 能力 | 说明 |
| --- | --- | --- |
| L1 | 规则计算 | 固定公式、阈值、红黄绿告警 |
| L2 | 情景模拟 | 用户调参、What-If、敏感性分析 |
| L3 | 启发式优化 | 贪心、排序、局部最优、规则调度 |
| L4 | 约束优化 | MILP、网络流、多目标优化、全局最优 |
| L5 | 自学习决策 | 预测模型、强化学习、自动策略迭代、在线评估 |

当前项目大致处于 L2-L3，部分模块具备 L4 的数据结构基础。下一阶段目标应是把时间套利、空间套利和全局调度逐步升级到 L4。

### 6.2 时间套利算法升级

输入：

- 当前价格、期货曲线、预测价格。
- 仓储成本、资金成本、损耗成本。
- 库存批次、库龄、部位、数量。
- 产能、库容、销售和深加工能力。

输出：

- 每个批次/部位的最佳持有周期。
- 最佳释放月。
- 持有收益和风险调整收益。
- 超龄风险和提前出库建议。

升级方式：

- 用真实期货曲线替代固定自然上扬曲线。
- 加入季节因子和历史回测。
- 加入资金上限、库容上限、库龄上限和合同约束。

### 6.3 空间套利算法升级

输入：

- 产地供给、销地需求、区域价格、冷链车辆、路线和运费。
- 车型、载重、时窗、装卸能力、温控要求。

输出：

- 货量分配。
- 路线和车辆计划。
- 鲜销、冻储、深加工分流。
- 利润、风险、执行复杂度。

升级方式：

- 短期使用贪心 + 约束修正。
- 中期使用最小费用最大流。
- 长期使用 MILP 统一求解多仓、多车、多时间窗和多产品。

### 6.4 金融套利算法升级

输入：

- 现货敞口、库存、期货合约、保证金、手续费、资金成本。
- 风控限额、套保比例、审批状态。

输出：

- 套保建议、合约数量、资金占用、追保风险。
- 压力测试、VaR/ES、最差损失。

升级方式：

- 增加每日盯市和保证金监控。
- 增加合规策略模板：卖出套保、买入套保、基差套利、跨期套利。
- 增加权限：只有授权角色可模拟或提交金融策略。

### 6.5 全局优化算法升级

建议将 `globalOptimization.ts` 拆为：

- `optimization/input.ts`：输入模型和校验。
- `optimization/preprocess.ts`：数据预处理、单位换算、出品率树。
- `optimization/heuristic.ts`：当前启发式方案。
- `optimization/solver.ts`：未来 MILP/OR 求解器接口。
- `optimization/explain.ts`：结果解释和角色动作。
- `optimization/backtest.ts`：历史回测。

优化目标：

- 最大化总利润。
- 最大化风险调整收益。
- 最小化亏损。
- 最小化超龄库存。
- 最大化订单满足率。
- 最小化资金占用。

硬约束：

- 出栏量、屠宰、分割、速冻、库容、深加工、销售、物流、资金、合同、食品安全。

软约束：

- 客户优先级、渠道偏好、部位策略、执行复杂度、风险偏好。

## 7. AI 建设方案

### 7.1 AI 角色定位

AI 不应直接替代业务负责人，而应承担：

- 数据摘要。
- 策略解释。
- 根因分析。
- 工单草案。
- 风险提示。
- 自然语言调参。
- 复盘报告生成。

### 7.2 Agent 分工

建议固定 5 类 Agent：

1. 经营决策 Agent：收益、风险、策略优先级。
2. 产能调度 Agent：屠宰、分割、速冻、仓储、深加工瓶颈。
3. 物流销售 Agent：跨区调拨、渠道消化、客户订单。
4. 风控合规 Agent：高风险、金融、食品安全、合同违规。
5. 现场执行 Agent：工单、回执、SLA、升级。

### 7.3 AI 输出治理

所有 AI 输出必须满足：

- JSON schema 固定。
- 有规则兜底。
- 有输入上下文快照。
- 有模型版本和 prompt 版本。
- 高风险建议不自动执行。
- 结果写入审计或决策记录。

建议新增：

- `ai_prompt_versions` 表。
- `ai_decision_runs` 表。
- `ai_evaluation_cases` 表。
- AI 输出评分：准确性、可执行性、风险覆盖、业务采纳率。

## 8. 权限、审批与风控方案

### 8.1 角色

建议角色分层：

- `viewer`：只读。
- `analyst`：模拟和保存方案。
- `strategist`：提交策略。
- `executor`：接收和回执工单。
- `manager`：审批中高风险策略。
- `admin`：配置规则、用户和系统。
- `finance_risk`：金融套利和资金策略审批。

### 8.2 审批规则

必须审批的动作：

- 高风险库存策略。
- 超过资金限额的收储。
- 超过库容阈值的调拨。
- 跨大区或跨境调拨。
- 任何金融套保或期货相关动作。
- 无合同或低价销售储备部位。
- 超时升级后的替代执行方案。

### 8.3 风控指标

必须监控：

- 库龄 90 天、120 天阈值。
- 库存损耗率。
- 产能利用率过低或过高。
- 资金占用和保证金占用。
- 策略预期收益和实际收益偏差。
- 外部价格源异常。
- 工单超时率。
- 数据延迟和缺失率。

## 9. 工程实施路线图

### 阶段 0：现状固化与文档统一（1 周）

目标：冻结当前能力边界，形成工程文档和验收基线。

交付物：

- 项目全景分析报告。
- 项目建设方案。
- API 清单。
- 数据表清单。
- 页面清单。
- 当前测试报告。

验收：

- `pnpm check` 通过。
- `pnpm test` 通过。
- 文档进入 `docs/`。

### 阶段 1：数据口径与真实数据底座（2-4 周）

目标：把模拟数据和真实数据边界说清楚，并接入第一批真实数据。

任务：

- 建立数据字典。
- 建立部位主数据。
- 接入库存批次、财务成本、产能、销售订单和行情。
- 为所有数据增加 `sourceStatus`、`updatedAt`、`sourceName`、`qualityScore`。
- 增加数据质量任务和异常告警。

验收：

- 关键页面显示数据来源和更新时间。
- 行情、库存、成本、产能至少各有一条真实链路。
- 数据缺失时有明确降级，不误标为实时。

### 阶段 2：核心闭环生产化（3-6 周）

目标：让 AI 决策、派单、回执、审计成为真实闭环。

任务：

- 工单绑定真实用户、角色、组织、联系方式。
- 工单状态机标准化。
- 企业微信/短信真实配置。
- 高风险审批流。
- 审计筛选和复盘页面。
- 工单 SLA 和超时升级。

验收：

- 一条 AI 策略可以从生成、审批、派单、回执、升级到审计完整走通。
- 通知失败有重试和记录。
- 审计可以按批次、人员、风险、时间查询。

### 阶段 3：算法升级与回测（4-8 周）

目标：从演示算法升级为可信算法。

任务：

- 建立历史回测数据集。
- 时间套利引入真实期货曲线和历史季节因子。
- 空间套利引入真实物流路线和车辆约束。
- 全局优化引入求解器接口。
- 金融套利增加合规边界和限额。
- 每个算法输出解释、约束和风险贡献。

验收：

- 每个核心算法有回测报告。
- 策略建议与实际收益可对账。
- 关键策略有压力测试。

### 阶段 4：页面重构与体验提效（3-5 周）

目标：降低维护成本，提升业务使用效率。

任务：

- 拆分超大页面。
- 抽取图表组件、调参组件、表格组件、策略卡片。
- 统一加载、错误、空状态。
- 增加 E2E 测试。
- 优化移动端现场角色体验。

验收：

- 关键页面可维护组件化。
- 页面首屏和交互性能达标。
- E2E 覆盖核心流程。

### 阶段 5：生产部署与运维治理（2-4 周）

目标：形成可长期运行的平台。

任务：

- CI/CD。
- 环境隔离：dev/staging/prod。
- 数据库迁移流程。
- 日志、指标、链路追踪。
- 外部行情源监控。
- 错误告警。
- 备份和恢复。

验收：

- 生产发布可回滚。
- 关键接口有 SLO。
- 数据库有备份恢复演练。
- 告警可到负责人。

## 13. 主要风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 真实数据接入慢 | 算法停留在模拟 | 先接 P0 数据域，所有模拟数据显式标识 |
| 数据口径不一致 | 决策不可用 | 建立数据字典和业务签核 |
| 外部行情源不稳定 | 行情和策略失真 | 多源冗余、缓存、fallback、告警 |
| 算法不可解释 | 业务不敢用 | 输出公式、约束、风险贡献和回测 |
| AI 幻觉 | 决策误导 | JSON schema、规则兜底、人工审批 |
| 现场不用工单 | 闭环断裂 | 移动端角色视图、SLA、责任人、培训 |
| 金融策略风险 | 合规和资金风险 | 财务风控审批、限额、盯市和止损 |
| 页面和路由过大 | 维护困难 | 模块拆分和组件化 |

## 14. 结论

CP-AI Brain 下一阶段的主线应是“真实数据 + 可解释算法 + 执行闭环 + 审计治理”。当前代码已经证明了业务路径可行，继续推进时应避免只增加展示功能，而要把每一个策略模块都做成可验证、可落库、可审批、可复盘的经营工具。

建议从 AI 决策派单闭环和真实库存/成本/产能数据接入开始，因为这两件事最能把平台从演示系统推向真实经营系统。随后再用回测和求解器升级时间套利、空间套利和全局优化，最终形成覆盖猪事业全产业链的生产级 AI 量化决策平台。
